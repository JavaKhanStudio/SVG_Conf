Comment ca marche

Chaque SVG de la galerie declare ses parametres dans une regle :root, et accole un commentaire /* @ws ... */ qui dit a la Black Room comment construire le bon controle. C'est tout. Le fichier reste un SVG valide ouvrable dans n'importe quel navigateur.

CSS dans le SVG :root { --bg-color: #1a1a2e; /* @ws color */ --shape-size: 50; /* @ws number min=10 max=100 */ --accent: #f09040; --accent-shadow: #7a4820; /* @ws mix=accent:-0.55 */ }

Compare place le SVG et la photo source cote a cote, Telecharger recupere ton SVG avec tes modifications, et Voir le code source sous l'illustration ouvre le fichier brut dans un nouvel onglet.

    Quand une IA utilise la Black Room

    La Black Room expose un vrai pipeline pour qu'un agent IA transforme une photo de telephone en SVG parametrique. Tout est decrit dans .claude/skills/svg-from-photo/SKILL.md (~200 lignes) : l'agent recoit des outils via le CLI svgw (backend Python port 5174), boucle sur les mesures, et corrige region par region.

    1
    Preprocess

    svgw preprocess ecrit 7 variantes (original, gray, otsu, adaptive, canny, bilateral, depth) dans .workshop/<svg>.refs/. Canny = outline, otsu = silhouette.

    2
    Trace

    svgw trace injecte un <g id="trace-ref"> cache dans le SVG cible. Le bouton "Trace" l'affiche en surcouche magenta.

    3
    Decider : trace ou hand-draft

    Line art / logos : vtracer direct en mode color, paths re-buckets par classe CSS. Photos avec profondeur : hand-draft parametrique par-dessus le trace en reference.

    4
    Posterise (stylisation)

    Photo vers illustration parametrique : boost saturation HSV x2.5 + k-means k=8 + vtracer. Puppy : 17900 → 279 paths, 20.7 Mo → 1.4 Mo, 8 regions nommees.

    5
    Measure

    svgw measure renvoie 3 scores : outline_iou, pixel_iou, edge_ssim. Passes optionnelles : subject_bbox, per_region_density, symmetry.

    6
    Revise + echantillonner couleurs

    Boucle sur la pire region, jusqu'a outline_iou ≥ 0.85 ou 5 passes. Puis svgw colors echantillonne une couleur par region nommee avec score de confiance.

    7
    SVGO workshop-safe

    Optimisation finale via tools/svgo.config.mjs. Reduction moyenne ~85% (stone 16 Mo → 2 Mo) avec drift metriques sous 0.01 IoU.

    8
    Text overlay

    Pour posters / logos : bucket les fills texte, auto-derive position/taille via bbox + cluster Y, puis injecte de vrais <text>. Le traced text se cache via --traced-text: none.

    Le pattern : le backend Python fournit les mesures ; l'agent est la boucle. L'interface ci-dessus est pour que l'humain regarde en direct et intervienne. Source complet : .claude/skills/svg-from-photo/SKILL.md.
