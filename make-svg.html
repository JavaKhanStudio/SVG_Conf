Recette : ton propre SVG parametrique

Le format que l'atelier comprend, en quelques lignes

Le principe

Tu declares tes parametres en variables CSS dans une regle :root, placee dans une balise <style> en tout debut du SVG. Tu reutilises ces variables via var(--ma-variable) dans les attributs. C'est tout : le SVG reste 100% valide, ouvrable dans Inkscape, dans une page web, ou dans l'atelier qui detecte les variables et fabrique les controles automatiquement.

Template minimal <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 200 200"> <style> :root { --bg-color: #1a1a2e; --shape-color: #4a90e2; --shape-size: 50; } </style> <rect width="100%" height="100%" fill="var(--bg-color)" /> <circle cx="100" cy="100" r="var(--shape-size)" fill="var(--shape-color)" /> </svg>

Inference automatique

Pas besoin d'annoter chaque variable. L'atelier devine le type a partir de la valeur :

Valeur ressemble aControle
#1a1a2e, rgb(...), redSelecteur de couleur
20, 1.5, 45deg, 100%Curseur numerique
true / falseCase a cocher
N'importe quoi d'autreChamp texte

Indices @ws (quand tu veux plus de controle)

Ajoute un commentaire /* @ws ... */ apres la valeur, sur la meme ligne, pour upgrader le controle. Les commentaires sont ignores par le navigateur, donc le SVG reste valide.

:root { --pupil-size: 20; /* @ws number min=5 max=50 step=1 */ --style-mode: soft; /* @ws select options=soft,sharp,neon */ --seed: 12345; /* @ws seed */ --orange: #f09040; --orange-shadow: #7a4820; /* @ws mix=orange:-0.55 */ --internal: 0.7; /* @ws ignore */ }
IndiceEffet
@ws number min=N max=N step=NCurseur avec plage et pas explicites.
@ws select options=a,b,cMenu deroulant avec les options listees.
@ws color / @ws textForce le type de controle si l'inference se trompe.
@ws mix=base-var:amountMarque la variable comme derivee d'une autre couleur. amount dans [-1, 1] : negatif assombrit, positif eclaircit. La variable suit automatiquement sa base.
@ws seedChamp numerique avec un bouton random.
@ws ignoreNe genere aucun controle (utile pour les helpers internes).

Exemple : deux couleurs liees

Le cas classique : une couleur principale et son ombre. Avec @ws mix=..., tu modifies une seule couleur et l'autre suit en temps reel.

<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 200 100"> <style> :root { --orange: #f09040; --orange-shadow: #7a4820; /* @ws mix=orange:-0.55 */ } .sun { fill: var(--orange); } .shadow { fill: var(--orange-shadow); } </style> <circle class="sun" cx="60" cy="50" r="40"/> <circle class="shadow" cx="140" cy="50" r="40"/> </svg>

Pieges a eviter

  • N'oublie pas :root. Les declarations doivent vivre dans une regle :root { ... }. Une variable declaree au top-level du stylesheet sans selecteur ne sera pas detectee.
  • Place la balise <style> en premier enfant du <svg>. Pas dans un <g>, pas dans un <defs>. Premier enfant, c'est plus lisible et c'est ce que l'atelier cherche.
  • Kebab-case pour les noms. --light-x, pas --lightX. Convention CSS, plus lisible.
  • L'indice /* @ws ... */ doit etre sur la meme ligne que la declaration qu'il annote, apres le point-virgule. Sur une ligne separee, il ne sera pas associe.
  • Ne reecris pas le SVG quand tu changes une valeur dans l'atelier. Les modifications sont appliquees en surcouche via des styles inline ; le fichier source garde ses defauts.
