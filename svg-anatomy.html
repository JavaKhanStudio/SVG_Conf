Anatomie d'un SVG

Comprendre la structure du code SVG

Structure de base

Un SVG est un document XML avec un element racine <svg> et des elements enfants qui decrivent les formes.

SVG <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 200 200" width="200" height="200"> <!-- Shapes go here --> <circle cx="100" cy="100" r="80" fill="#0752CF" /> </svg>

Le ViewBox

L'attribut viewBox definit le systeme de coordonnees interne du SVG. Il permet de creer des graphiques responsifs qui s'adaptent a n'importe quelle taille.

viewBox viewBox="min-x min-y width height" <!-- Example: viewBox="0 0 100 100" --> <!-- Creates a 100x100 coordinate space --> <!-- The SVG scales to fit its container -->

La balise Path

L'element <path> est le plus puissant du SVG. Il peut dessiner n'importe quelle forme avec des commandes de deplacement.

Commandes principales : M (deplacer), L (ligne), H (horizontal), V (vertical), C (courbe cubique), Q (courbe quadratique), A (arc), Z (fermer)

Cote IA : ces commandes (M, C, Q, L...) sont exactement ce qu'un LLM ecrit quand on lui demande de dessiner. Une fleur, un coeur, un logo : tout devient une chaine de caracteres comme d="M30,45 C30,45 10,30 10,18...". Pour aller plus loin, voir les courbes de Bezier.

Stylisation

Les elements SVG peuvent etre styles avec des attributs, du CSS inline, ou des feuilles de style externes. Combine avec des variables CSS, le meme SVG devient parametrable a la volee, c'est exactement ce que fait la Black Room.

Attributes <rect fill="#0752CF" stroke="#BA0101" stroke-width="3" opacity="0.8" rx="10" /> <!-- Or with CSS --> <style> rect { fill: #0752CF; stroke: #BA0101; transition: fill 0.3s; } rect:hover { fill: #BA0101; } </style>

Recette : ton propre SVG parametrique

Le principe

Tu declares tes parametres en variables CSS dans une regle :root, placee dans une balise <style> en tout debut du SVG. Tu reutilises ces variables via var(--ma-variable) dans les attributs. C'est tout : le SVG reste 100% valide, ouvrable dans Inkscape, dans une page web, ou dans la Black Room qui detecte les variables et fabrique les controles automatiquement.

Template minimal — modifie a gauche, regarde a droite

Inference automatique

Pas besoin d'annoter chaque variable. La Black Room devine le type a partir de la valeur :

#1a1a2e, rgb(...), redSelecteur de couleur
20, 1.5, 45deg, 100%Curseur numerique

N'importe quoi d'autreChamp texte

Indices @ws (quand tu veux plus de controle)

Ajoute un commentaire /* @ws ... */ apres la valeur, sur la meme ligne, pour upgrader le controle. Les commentaires sont ignores par le navigateur, donc le SVG reste valide.

:root { --pupil-size: 20; /* @ws number min=5 max=50 step=1 */ --style-mode: soft; /* @ws select options=soft,sharp,neon */ --seed: 12345; /* @ws seed */ --orange: #f09040; --orange-shadow: #7a4820; /* @ws mix=orange:-0.55 */ --internal: 0.7; /* @ws ignore */ }
IndiceEffet
@ws number min=N max=N step=NCurseur avec plage et pas explicites.
@ws select options=a,b,cMenu deroulant avec les options listees.
@ws color / @ws textForce le type de controle si l'inference se trompe.
@ws mix=base-var:amountMarque la variable comme derivee d'une autre couleur. amount dans [-1, 1] : negatif assombrit, positif eclaircit. La variable suit automatiquement sa base.
@ws seedChamp numerique avec un bouton random.
@ws ignoreNe genere aucun controle (utile pour les helpers internes).

Exemple : deux couleurs liees

Le cas classique : une couleur principale et son ombre. Avec @ws mix=..., tu modifies une seule couleur et l'autre suit en temps reel.

Pieges a eviter

  • N'oublie pas :root. Les declarations doivent vivre dans une regle :root { ... }. Une variable declaree au top-level du stylesheet sans selecteur ne sera pas detectee.
  • Place la balise <style> en premier enfant du <svg>. Pas dans un <g>, pas dans un <defs>. Premier enfant, c'est plus lisible et c'est ce que la Black Room cherche.
  • Kebab-case pour les noms. --light-x, pas --lightX. Convention CSS, plus lisible.
  • L'indice /* @ws ... */ doit etre sur la meme ligne que la declaration qu'il annote, apres le point-virgule. Sur une ligne separee, il ne sera pas associe.
  • Ne reecris pas le SVG quand tu changes une valeur dans la Black Room. Les modifications sont appliquees en surcouche via des styles inline ; le fichier source garde ses defauts.
