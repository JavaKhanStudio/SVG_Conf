Acte 3

Creation SVG complete

Ce que l'IA ne fait pas (encore) toute seule

Une carte a deux faces

Autre projet, autre jour. Je voulais permettre aux gens de tirer une carte de tarot. Probleme : selectionner directement une image generee par IA me semblait trop lache. Une carte, c'est deux faces.

Logique : le devant montrerait l'arcane (AI-generated), et la carte se retournerait au clic grace a un composant CSS 3D. Mais que mettre au dos ? Encore une image IA ? Non, je voulais quelque chose de personnel, un SVG qui puisse evoluer avec les choix de l'utilisateur.

L'IA est excellente pour imaginer une face avant. Pour le dos je voulais coder un langage visuel coherent, reutilisable, parametrable. Bienvenue dans un SVG de 451 lignes.

La carte

Joue avec les couleurs et les themes. Chaque theme redessine les quatre coins avec un autre set d'ornements : Chair (organique), Matiere (geometrique) ou Ame (mystique).

Anatomie de la carte

Six techniques SVG qui composent la carte. Chacune s'utilise independamment et se combine avec les autres, c'est ce qui fait la richesse du format.

1 2 3 4 5 6
1
Variables CSS

Un seul :root { } dans le SVG expose cinq couleurs. Tout ce qui reference var(--couleur) se met a jour en temps reel, sans regenerer le fichier.

2
<defs> et <use>

Une forme est definie une fois dans <defs> puis instanciee avec <use href="#id"> avec un transform different. La meme brique, rayon ou triangle est reutilise des dizaines de fois sans gonfler le fichier.

3
Gradients

Quatre gradients superposes (radiaux pour l'iris et la sclere, lineaires pour les ornements) construisent la profondeur. L'iris seul combine 3 stops avec opacites variables.

4
Filtres animes

Les chaines feGaussianBlur + feMerge donnent le halo. L'element <animate> de SMIL fait pulser la valeur stdDeviation entre 2 et 7 sur 3 secondes, en boucle.

5
Clip paths

La pyramide est un triangle, rempli de briques rectangulaires. Un <clipPath> au contour triangulaire decoupe les briques qui depassent : on dessine un rectangle bete, le clip fait le reste.

6
Symetrie via transform

L'ornement d'un coin est dessine une fois, puis duplique aux trois autres avec scale(-1,1), scale(1,-1), scale(-1,-1). Change un coin, tous les autres suivent.

Et si on l'animait ?

Meme carte, mais avec quelques animations CSS : une rotation lente sur le motif de rayons, une derive de teinte sur la palette, une levitation discrete. Le SVG, c'est du texte qui peut tout faire.
